# Speech Context Corrector

Use this skill when Titus dictates through speech-to-text and wording may contain accent/STT errors. Correct obvious misspellings and misheard project names using known AI OS context before writing docs, portfolio copy, memory, or code.

## High-confidence corrections

- Nollarita / NOLA Rita / Nollar reader → NOLA Voice Reader
- Darkflow / dog flow → DocFlow
- Pipeline Serium / Pipeline Serum / Serium → Pipeline CRM
- Darkflow Pipeline Serium → DocFlow + Pipeline CRM
- skew → skill, when discussing reusable AI behavior
- spallings → spellings
- talk to speech → speech-to-text or dictation
- whisper my idea / whisper pro → Whisper My Idea Pro
- command centre → Command Center
- side brain / system brain → AI OS Brain

## Rules

1. Preserve Titus's intent and tone.
2. Correct high-confidence STT errors silently.
3. For public-facing copy, use canonical project names from repo/context instead of raw dictated text.
4. If unsure and the spelling affects public copy, say what you interpreted briefly.
5. Never tease, over-explain, or make the correction the focus.
